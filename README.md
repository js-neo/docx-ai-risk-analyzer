# docx-ai-risk-analyzer

Local privacy-friendly analyzer for `.docx` academic documents.

The tool detects linguistic, structural, and statistical risk markers that may require editorial review. It does **not** prove that a text was generated by AI.

## Current status

The project currently includes:

- FastAPI backend for DOCX upload and analysis
- Next.js frontend for DOCX upload and result visualization
- heuristic text analysis pipeline
- API smoke tests
- environment-based frontend API URL configuration

## Core pipeline

1. Upload `.docx`
2. Extract readable text from paragraphs and tables
3. Split text into analysis blocks
4. Analyze linguistic and structural markers
5. Calculate risk level for each block
6. Show summary and block-level results in the web interface

## Backend API

### Health check

```text
GET /api/v1/health
```

Example response:

```json
{
  "status": "ok"
}
```

### Analyze DOCX

```text
POST /api/v1/analyze
```

The endpoint accepts a multipart form field named `file`.

Example response structure:

```json
{
  "status": "analyzed",
  "document": {
    "filename": "case1.docx",
    "content_type": "application/octet-stream",
    "size_bytes": 43705,
    "characters": 16459,
    "words": 1975,
    "paragraphs": 79,
    "blocks": 7
  },
  "summary": {
    "overall_risk": "low",
    "total_risk_score": 9,
    "high_risk_blocks": 0,
    "medium_risk_blocks": 0,
    "low_risk_blocks": 7,
    "cliches_total": 0,
    "abstract_words_total": 23
  },
  "blocks": [],
  "limitations": []
}
```

## Local development

### Backend

```bash
cd apps/api
uv sync --dev
uv run uvicorn --app-dir src docx_ai_risk_api.main:app --reload
```

Backend URL:

```text
http://localhost:8000
```

API docs:

```text
http://localhost:8000/docs
```

### Frontend

Create local environment file:

```bash
cp apps/web/.env.example apps/web/.env.local
```

Run frontend:

```bash
pnpm --filter web dev
```

Frontend URL:

```text
http://localhost:3000
```


The local `.env.local` file is ignored by Git. Use `.env.example` as a template.

## Limitations

- The tool does not prove that a text was generated by AI.
- The result is an editorial risk assessment based on linguistic and structural markers.
- Academic texts may produce false positives because of formal style and repeated terminology.
